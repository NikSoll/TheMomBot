init.py
стандарт.

main4.py
стандарт.

config4.py
опросы с разными типами вопросов.
для single и multiple нужны options.
для scale нужны min и max.
можно миксовать типы в одном опросе.

handlers4.py
самый сложный файл. обрабатывает 4 типа вопросов по-разному.
для multiple вопросов нужно хранить временные выбранные ответы (multiple_answers).
перед сохранением собираем все ответы в список словарей.
ответы сохраняются в гугл таблицу как json строка.

keyboards4.py
для multiple вопросов кнопки меняются ("✅" выбранные варианты).
нужно передавать selected список для подсветки.
кнопка "готово" для завершения multiple выбора.

states4.py
как в квизе - два состояния.

database4.py
сохраняет ответы в json формате. потом можно распарсить и анализировать.
важно чтоб json не сломался от спецсимволов - uses ensure_ascii=False.

utils4.py
форматирование списка опросов.
в результатах не показывает сами ответы, только названия опросов.

важно помнить
json поле в гугл таблице может быть длинным.
при большом количестве ответов лучше хранить в отдельной таблице.
